Summary

Month-end used to take several days and led to frequent disputes. We built a consolidated staging model (clients, invoices, receipts) and a repeatable transformation pipeline in Power Query with clear data contracts between steps.

A VBA generator produced signed statements per client, embedded anomaly notes (missing POs, unmatched receipts), and stored files in region-based folders. Optional Power BI views surfaced trends and aging.

  • Consistent, auditable statements delivered automatically
  • Anomaly & exception sheet for targeted cleanup
  • ~80% time saved at month-end

Problem

Manual sorting and statement creation were error-prone and slow. No single source of truth existed.

Approach

Normalized sources with Power Query; applied parameterized transforms; built a VBA packager to export per-client PDFs with consistent naming and sign-offs.

Architecture / Stack

Excel + Power Query, VBA, optional Power BI; distribution via shared folders or SharePoint/Drive.

Results

  • Fewer disputes thanks to a clear audit trail
  • Predictable delivery cadence and standardized outputs
